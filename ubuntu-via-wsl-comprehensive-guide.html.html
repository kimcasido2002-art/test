How to Use Ubuntu on Windows with WSL – A Simple Guide

Introduction

Ubuntu on Windows through the Windows Subsystem for Linux (WSL) looks like it’s changing the way people who use Windows work with Linux. It gives a bridge between Windows and a real Ubuntu system. No more double boot or big virtual machines. For students, developers and even hobbyists this mix of ease and speed can feel like a shortcut. It may seem too good to be true, but many already swear by it.

What does “Ubuntu on WSL” actually mean?

Ubuntu on WSL is basically a full Ubuntu system installed inside Windows 10 or Windows 11. You get a regular looking Linux terminal, the usual GNU tools and you can run Linux programs side by side with Windows apps. In practice you are running Ubuntu without ever leaving Windows. That lets you do cross platform work that used to feel clunky.

WSL vs. WSL 2 – Quick Compare

  • WSL 1 – First version. It translates Linux calls into Windows calls. It works, but file access can be slow, especially with big projects.
  • WSL 2 – Adds a tiny virtual machine that runs a real Linux kernel. Files move faster, system calls work better and Windows resources are more reachable. Most people now pick Ubuntu on WSL 2 because it feels more “real” Linux.

Installing Ubuntu on WSL

You can install it with PowerShell or the Microsoft Store – whichever you prefer.

Step by Step Installation

  1. Enable the WSL feature
    - Open PowerShell as Administrator.
    - Type wsl --install and hit Enter. This one liner turns on WSL and, on Windows 10 2004+ or Windows 11, pulls Ubuntu as the default distro.
  2. Restart your PC
    - A quick reboot lets the new subsystem start up properly.
  3. Launch Ubuntu
    - Find “Ubuntu” in the Start menu, click it, then follow the prompts to make a Unix username and password. That finishes your Ubuntu user set up.

Alternate way: Open Microsoft Store, search “Ubuntu”, click Get, wait for it to download, then run it and do the same username/password steps.

Using Ubuntu on WSL – Tips and Tricks

Running Linux GUI apps

New Windows updates bring WSLg, which lets graphic Linux programs run out of the box. To use it:

  • Run wsl --update to get the latest runtime.
  • Follow Microsoft’s short guide on WSL GUI apps (link below).
  • Then you can start GUI tools like gedit, Firefox or any X based program right from the Ubuntu terminal.

Accessing Linux files from Windows

Your Ubuntu files are reachable in Windows Explorer via the path \\wsl$\Ubuntu\home\your_username. Drag and drop works, so you don’t have to copy files back and forth constantly.

Keeping Ubuntu up to date

Just like normal Ubuntu, you should update often. Inside Ubuntu run:

  • sudo apt update – refreshes the package list.
  • sudo apt upgrade – installs newer versions of packages.

Running these twice a week keeps security fixes coming.

Why use Ubuntu on WSL? – Pros and Cons

  • Smooth integration – Linux and Windows tools can live together. You can call gcc, npm or docker while still using PowerShell scripts.
  • Time saver for developers – Open source toolchains, container work and web stacks that normally need Linux run instantly.
  • No need to reboot – Skip the hassle of dual booting; stay in one OS.
  • Light on resources – Uses far less RAM and CPU than a full virtual machine.
  • GUI support – You can launch graphic Linux apps without leaving Windows.

But it isn’t perfect. Some heavy duty Linux workloads still run slower than on a native machine. Network tricks that need low latency can misbehave. And a few Windows only file system quirks still bite new users.

Frequently Asked Questions

Q1: Can I develop software with Ubuntu on WSL?
Yes. Most Linux dev tools work: Python, Node.js, C/C++, Java and Docker all run fine.

Q2: How do I update Ubuntu on WSL?
Just type sudo apt update && sudo apt upgrade in the Ubuntu terminal. That updates the index and upgrades everything in one go.

Q3: Is WSL 2 faster than WSL 1?
Generally yes. The real Linux kernel in WSL 2 gives better I/O speed, especially when dealing with many files.

Q4: Can I run a full Linux desktop?
With WSLg you can start a full desktop like Xfce or GNOME, though it may feel a bit “inside” a Windows window.

Q5: How do I remove Ubuntu from WSL?
Open PowerShell and type wsl --unregister Ubuntu. You can also uninstall it from Windows Settings → Apps… Features.

Conclusion

Ubuntu on Windows through WSL merges Linux power with Windows familiarity. Using WSL 2 you get near native speeds and the new WSLg adds graphical apps. It removes the biggest annoyances of dual booting or big VMs. Still, remember it isn’t a perfect copy of a real Linux box – some corner cases bite. Play around with tutorials, try GUI programs, and you’ll probably find a workflow that fits your needs.
